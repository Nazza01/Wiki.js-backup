For an existing code repo that you want to resubmit (42 School)
You may want to re-submit the code and make a few changes.

In my case I have libft to still get marked and as such, need to make changes to my code then re-submit again.

I have my existing folder setup as so:

$ ls ~/42Git/libft
		.git		Old Git repo folder
		.gitignore	Git ignore file
		*.c			Source files
		*.h			Header files
		Makefile	Source Makefile
		........	Other files

If I make any changes to my code and the repo is no longer able to be committed to (after submitting subscribe to a defence)
Rather than copy all the old files over. We can re-use the existing code and re-commit it using the below commands.

This all assumes that you have the git repository link after clicking re-try. 

Step 1: Remove the hidden old .git repository folder

I need to remove the old .git folder like so

$ rm -rf .git					-Remove the .git folder recursively and forcefully

Step 2: Intialise the current directory (e.g. libft) as a local git repository.
Make sure you are in the correct directory

pwd: ~/42Git/libft				-Check the current directory
git init						-Create the .git folder to start the git repo
git remote add origin <url>		-Link the local .git folder to the remote location
git add . 						-Add the local files in the current directory to the files to be commited

Before you submit a commit message before pushing your code up you need to set up your git configuration with your email and name used for the git commit logs/details:

git config user.email "you@example.com		-Add your details in here so logs can be easily traceable
git config user.name "Your Name"			-Add your name for logs' sake
git commit -m "message"						-Add a message to your logs for reference

You will need to push the intial commit to the origin master branch so the future commits will know where to go

git push --set-upstream origin master		-Make the current commit branch a link to the origin master on the server
